# 第十三章：学硬科学/数学——完整案例

## 开篇：一个工程师想真正理解信息论

假设你是一个有多年编程经验的工程师。

你熟悉代码。你知道一个系统如何拆成模块，知道接口为什么要稳定，知道缓存为什么能提高性能，知道网络重试为什么要加退避，知道日志为什么要结构化。你见过足够多的线上问题，也知道很多抽象原则最后都会落到延迟、吞吐、可维护性和故障恢复上。

现在你想学信息论。

不是为了背几个公式，也不是为了把“熵”这个词挂在嘴边。你真正想要的是判断力：当面对一个系统、一次沟通、一段日志、一个压缩问题、一个学习问题时，你能不能判断“这里的不确定性是否可以被量化？我能不能用熵、编码、噪声、信道这些概念做决策？”

比如，一个推荐系统为什么要探索？一个日志字段为什么有些很有信息量，有些只是噪声？一个模型为什么能压缩数据就说明它抓住了结构？一个团队沟通为什么会因为上下文缺失而失真？这些问题不全是信息论问题，但信息论会给你一套非常硬的底层语言。

问题是，信息论不像学一门新编程语言。

学匀加速运动，你可以问："位移公式和匀速运动有什么不同？"学电磁学，你可以问："电场力分析和重力分析有什么不同？"学化学平衡，你可以问："平衡常数和物理中的比例常数有什么不同？"

这些都能做差异映射，因为你已经有接近的已知框架。

但信息论不一样。你不能认真地说：“信息论里的熵和编程里的 try-catch 有什么不同？”这个映射太弱。try-catch 是异常控制流，熵是对不确定性的度量。它们都和“意外”有关，但这种相似只停在词面上，不能支撑学习。

这就是硬科学学习的特殊挑战：你经常没有现成的旧框架可以挂靠。它不是另一个工具，而是另一套基元。你不能总靠类比进入，有时必须先造一个粗糙基线，再用后续学习不断修正它。

本章就走一遍完整案例：一个有编程背景的人，如何学习信息论基础。

## 第一步：HLT 评分

面对硬科学，很多人会先害怕。

公式多，符号密，教材冷，例子少。翻开第一页，看到概率、对数、期望、极限，心里立刻出现一个声音：这东西是不是太硬？我是不是应该先把数学补完？我是不是没有必要学这么深？

先别急着投入，做 HLT 评分。

H 是半衰期。

信息论的半衰期极长。1948 年，香农发表《通信的数学理论》，奠定现代信息论基础。七十多年过去，通信系统、计算机、互联网、机器学习、压缩算法都发生了巨大变化，但熵、互信息、信道容量、编码定理这些核心概念并没有过时。

你今天理解“熵是对不确定性的度量”，十年后它仍然成立。你今天理解“压缩利用的是分布不均匀性”，十年后它仍然成立。你今天理解“噪声信道中可靠传输需要冗余”，十年后它仍然成立。所以 H 给 10 分。

L 是杠杆。

信息论的杠杆不在于让你马上多写一个功能，而在于它会改变你理解很多事情的方式。

理解熵之后，你会重新看待“信息量”。一句话长，不代表信息量大；一个字段占空间，不代表它有价值。真正有信息量的是它减少了多少不确定性。一个总是取同一个值的字段，即使存了一百万行，也几乎不提供新信息。一个少见但能区分关键状态的字段，可能非常有价值。

理解编码之后，你会重新看待表达。好的编码不是把所有东西平均对待，而是把高频事件用短码表示，把低频事件用长码表示。自然语言、表达方式、日志格式、界面布局、团队术语，里面都有类似结构。

理解噪声之后，你会重新看待沟通和学习。沟通不是“我说了，对方就收到了”。中间有信道，有噪声，有丢失，有误码，有上下文不一致。学习也不是“我看了，就理解了”。输入材料经过注意力、记忆、已有框架和误解的信道，最后留下来的东西可能已经变形。

所以 L 给 7 分。

T 是迁移性。

信息论最初服务于通信问题，但今天你可以在数据压缩、密码学、机器学习、统计推断、神经科学、语言学、组织沟通、产品设计里看到它的影子。迁移不是把公式到处乱套，而是多一个抽象视角：这个系统里的不确定性在哪里？谁在减少不确定性？哪些信号有信息量？哪些只是噪声？有没有冗余？有没有压缩？

所以 T 给 9 分。

```text
HLT = H × L × T = 10 × 7 × 9 = 630
```

630 是核心杠杆。

这意味着你不应该只看几篇科普文章，也不应该只收藏几个公式。信息论值得走全面精通五层。不是研究生级别的全面精通，而是对工程师有意义的精通：能跑通计算，能拆解数据流，能做参数实验，能探索边界，最后能把概念封装成自己的判断语言。

## 第二步：建基线

如果这是学一门编程语言，下一步通常是差异映射。

你学电磁学，就拿力学做基线。你学化学平衡，就拿物理守恒做基线。你学解析几何，就拿平面几何做基线。你问"它和我已经会的东西有什么不同"，学习就启动了。

但学信息论时，这一步会遇到硬边界。

你没有一个现成的信息论框架。

你可能知道一点概率，知道 log 是对数，知道二进制，知道压缩文件会变小，知道网络传输会出错。你也可能听过“熵代表混乱程度”。但这些碎片拼不成框架。

更麻烦的是，很多已有类比会误导你。

比如“熵就是混乱程度”。这句话在某些语境下有用，但对信息论入门来说不够好。因为你真正需要理解的是：熵度量的是一个随机变量平均带来的不确定性。一个序列看起来乱，不等于它的熵一定高；一个系统看起来有序，也不等于没有信息。

再比如“压缩就是去掉重复”。这句话也有帮助，但不完整。压缩不只是去掉字面重复，更深层的是利用概率分布的不均匀性。一个完全随机的比特串没有明显重复，但它也很难压缩，因为每个位置都接近最大不确定性。

所以要降级：先建立粗糙基线。

第一轮学习不要以“彻底理解”为目标。目标更低：建立词汇表和直觉。

你可以选一本最薄的信息论入门书，只读第一章。不要从厚教材开始，不要一上来挑战信道编码定理的证明，也不要试图把所有概率论基础一次补齐。你只是想知道这个领域在谈哪些对象：信息、熵、随机变量、概率分布、编码、信源、信道、噪声、冗余、压缩、互信息、信道容量。

同时配一本更轻的科普读物，比如《编码的奥秘》。它不是严格的信息论教材，但它能帮你把“编码”“二进制”“冗余”“纠错”“通信”这些概念和计算机经验连接起来。硬科学学习最怕概念悬空，科普读物的价值就是先给概念找地面。

这轮阅读要故意放松。看到公式，不急着证明。看到定义，不急着背。看到不懂的推导，先标记。你现在不是在建摩天楼，而是在搭脚手架。脚手架粗糙、临时、甚至有些地方不稳，都可以接受。没有脚手架，你后面连往哪里用力都不知道。

读完第一轮之后，你要产出一张"我最熟悉的概念和信息论新概念"的映射草稿。

| 熟悉的概念 | 信息论里的新概念 | 当前理解 |
| --- | --- | --- |
| 一个情况的各种可能结果 | 随机变量 | 可能的结果是一个可取不同值的对象，但信息论更关心它的概率分布 |
| 二分判断 | 二进制编码 | 每次二分都像是在问一个 yes/no 问题 |
| 纠错机制 | 纠错编码 | 出错不可怕，关键是能不能用冗余恢复 |
| 预先整理 | 压缩 | 都利用重复和可预测性，但优化目标不同 |
| 观察维度 | 随机变量 | 维度有取值分布，分布越不均匀，平均不确定性越低 |

这张表不一定准。

"预先整理 = 压缩"就不准确。预先整理保存高成本计算或访问的结果，用空间换时间；压缩是用更短表示保存同样信息，用计算和编码结构换空间。它们都利用结构，但不是同一件事。

"纠错机制 = 纠错编码"也不准确。纠错机制通常是在问题发生后决定如何恢复或降低影响；纠错编码是在传输前加入冗余，使接收端在一定噪声范围内恢复原始信息。一个偏控制流，一个偏数学设计。

但这没有关系。

粗糙基线的价值不在于准确，而在于让你开始能提问。没有这张表时，“熵”只是陌生词。现在你至少能问：熵是不是类似一个字段的可预测程度？如果日志字段永远是 `success`，它是不是熵很低？如果字段均匀分布在一千个用户 ID 上，它是不是熵很高？

建基线还有一个关键动作：写临时定义。

```text
信息量：一个事件发生后，它减少了多少不确定性。
熵：还没看到结果前，我平均有多不确定。
编码：把事件映射成符号序列。
压缩：利用事件分布的不均匀性，让常见事件用更短表示。
噪声：传输或观察过程中引入的不确定扰动。
冗余：为了抗噪声或便于恢复而额外加入的信息。
信道：输入到输出之间可能失真的传递机制。
```

这些定义还不严谨，但已经能支撑下一步学习。你现在有了最小词汇表，有了粗糙类比，有了几个能落到编程经验上的例子。差异驱动终于可以启动了。

## 第三步：差异映射

基线建立之后，开始问黄金三问。

第一问：信息论和我熟悉的哪个领域最像？

如果只看表面，它像压缩算法、网络协议或错误恢复机制。但从更深一层看，信息论在“决策”层面的结构最像贝叶斯推理。

贝叶斯推理关心的是：观察到新证据后，我对世界的信念如何更新。信息论关心的是：观察到一个信号后，我的不确定性减少了多少。二者都把“知道”和“不知道”变成可以计算的东西。

比如排查一道物理题。一开始，你不知道问题来自受力分析、能量守恒、运动方程还是边界条件。此时不确定性很高。你看到第一个信号：题目涉及电场。它排除一部分可能。你看到第二个信号：带电粒子初速度不为零。它又减少一部分不确定性。每个有效观察都在压缩解题空间。

贝叶斯会问：这些证据如何改变各个假设的后验概率？

信息论会问：这些信号减少了多少不确定性？哪个信号最有信息量？哪些日志字段看似很多，其实不能区分状态？

第二问：信息论做了什么，不做什么？

信息论做了一件核心事情：量化不确定性。

它不只是说“这个系统很随机”，而是问：随机变量的分布是什么？每个事件的概率是多少？平均需要多少比特才能描述它？接收一个符号后，不确定性减少了多少？在有噪声的信道中，最多能可靠传多少信息？

但信息论不替你消除不确定性。

熵能告诉你不确定性有多大，但不会自动告诉你下一次具体发生什么。信道容量能告诉你理论上能传多少信息，但不会自动给你工程实现。互信息能告诉你两个变量之间共享多少信息，但不会自动解释因果关系。

第三问：这些差异在已有框架中有等价物吗？

编码可以和数据格式、协议、序列化建立联系。纠错编码可以和重试、校验、幂等、校验和建立联系。噪声可以和网络错误、观测误差、沟通失真建立联系。

但熵没有直接等价物。

你可以把它类比成“平均惊讶程度”，也可以把它类比成“字段的不确定性”，还可以把它类比成“最优压缩后的平均长度下限”。这些都对，但都只是侧面。

熵是一个新基元。

这时候要停止强行类比。硬科学学习中，一个危险习惯是把所有新概念都塞进旧框架。类比能帮你入门，也会限制你。遇到真正的新基元，要允许它独立存在。

## 第四步：精通五层快步通过

### Level 1：跑通

第一层只追求一件事：让概念动起来。

不要从证明开始。先让 Agent 写一个最小脚本，计算一个简单序列的熵。

```
from collections import Counter
from math import log2


def entropy(xs):
 if not xs:
 raise ValueError("empty sequence has no empirical distribution")

 total = len(xs)
 counts = Counter(xs)
 h = 0.0

 for count in counts.values():
 p = count / total
 h -= p * log2(p)

 return h


print(entropy([0, 0, 0, 0]))
print(entropy([0, 1, 0, 1]))
print(entropy([0, 1, 2, 3]))
print(entropy([0, 0, 0, 1]))
```

输出大致是：

```text
0.0
1.0
2.0
0.811...
```

全是 0 的序列，熵是 0。因为没有不确定性。0 和 1 各一半，熵是 1。因为每次需要回答一个二选一问题，平均需要 1 个 bit。0、1、2、3 均匀出现，熵是 2。因为四种等概率结果，需要 2 个 bit 区分。三个 0 一个 1，熵约 0.811，因为它不是完全确定，但也没有均匀二元变量那么不确定。

跑通之后，熵就不再是玄学词。它是可计算的。

### Level 2：拆解

第二层要拆数据流。

输入是什么？

对熵计算来说，真正的输入是一个随机变量的经验样本，或者一个概率分布。列表只是样本形式。你用 `Counter` 统计频率，是在用样本估计分布。

做了什么？

```text
[0, 0, 0, 1] -> {0: 3, 1: 1}
0 -> 3/4
1 -> 1/4
H = -(3/4) * log2(3/4) - (1/4) * log2(1/4)
H = 0.811...
```

输出意味着什么？

它不是“这个列表有多少信息”。更准确地说，它是在这个经验分布下，抽取一个符号时的平均不确定性，单位是 bit。

这个区分非常重要。如果你有一个长度为 1000 的列表，熵仍然可能是 1 bit。因为熵描述的是单次抽样的平均不确定性，不是整个文件的总大小。整个序列的信息量和长度有关，但单符号熵是分布属性。

为什么用 `log2`？

因为我们用 bit 做单位。一个二选一问题，如果两种结果等概率，正好需要 1 个 bit。四种等概率结果需要 2 个 bit。八种等概率结果需要 3 个 bit。对数把“可能状态数”转换成“需要多少二进制问题来区分”。

### Level 3：参数实验

第三层是硬科学学习里最关键的一层。公式到直觉之间有一条沟，参数实验就是桥。

你要系统改变序列分布，先预测，再运行。

```
entropy([1, 1, 1, 1, 1, 1, 1, 1])
entropy([0, 1] * 50)
entropy([0, 1, 2, 3] * 25)
entropy(list(range(8)) * 20)
```

预测分别是 0、1、2、3。

这里出现一个模式：

```text
均匀分布下，H = log2(可能结果数)
```

这句话比背公式更有用。它把熵和“区分多少种可能”直接连起来。

然后做偏斜分布。

```
entropy([0] * 90 + [1] * 10)
entropy([0] * 99 + [1] * 1)
entropy([0] * 50 + [1] * 50)
```

50/50 的二元分布熵最高，是 1。90/10 更低。99/1 更低。因为结果越来越可预测。

再做一个工程化实验：

```
status_codes = [200] * 950 + [404] * 30 + [500] * 20
user_agents = ["Chrome"] * 400 + ["Safari"] * 250 + ["Edge"] * 150 + ["Firefox"] * 120 + ["Other"] * 80
feature_flags = ["on"] * 1000
```

`feature_flags` 熵为 0，因为它没有区分能力。`status_codes` 熵不高，因为大多数都是 200。`user_agents` 熵更高，因为分布更分散。

这会改变你看日志和指标的方式。一个字段有没有价值，不只看它有没有被记录，而要看它能不能减少不确定性。如果一个字段永远不变，它对排查问题几乎没有信息量。

### Level 4：边界探索

第四层要故意制造失败。硬科学概念最容易在边界处暴露真实含义。

第一个边界：空序列。

```
entropy([])
```

很多人会说熵是 0，因为“没有东西”。但这不严谨。空序列没有经验分布。你没有任何样本，无法估计每个事件的概率。熵是定义在概率分布上的，不是定义在“空无”上的。工程实现里更合理的做法是抛出错误，或者要求调用者显式提供先验分布。

第二个边界：全 1 序列。

```
entropy([1, 1, 1, 1])
```

答案是 0。

这次不是因为没有数据，而是因为分布完全确定。样本告诉你，这个经验分布里只有一个事件，概率为 1。`-1 * log2(1) = 0`。没有不确定性，所以熵为 0。

空序列和全 1 序列非常容易被初学者混淆。一个是没有分布，一个是分布退化为确定事件。前者不能算，后者能算且为 0。

第三个边界：每个值都只出现一次。

```
entropy(list(range(1000)))
```

结果大约是 `log2(1000)`，约 9.97。

这是不是说明这个序列“信息量巨大”？要小心。如果这些样本真来自一个有 1000 个等概率取值的随机变量，那么单次抽样熵接近 10 bit。可是如果这只是一个人为生成的递增序列，真实生成规则非常简单，那么把它当作“1000 个等概率符号”来算经验熵，就会误导你。

这触及更深的问题：经验分布依赖你如何建模。同一段数据，可以按字符建模、按单词建模、按字段建模、按生成程序建模。公式很硬，但选择随机变量仍然需要判断。

### Level 5：封装

第五层要把概念压缩成自己的语言。

```text
熵 = 我对下一个事件有多不确定。
量化后，它表示在最优编码下，平均需要多少 bit 来描述这个事件。
```

这句话有三个关键点。

第一，熵面向“下一个事件”，也就是随机变量的一次取值。它不是简单描述已有文件有多大。

第二，熵度量“不确定性”。如果结果确定，熵为 0；如果结果均匀分布在更多可能之间，熵变大；如果分布偏斜，熵下降。

第三，熵和编码连接。它不只是哲学上的不确定性，而是和“平均需要多少 bit 表示”有硬关系。

有了这层封装，你就能在工程场景里使用它。

看日志系统，你会问：哪些字段熵高？哪些字段熵低？熵高的字段是不是有用，还是只是随机噪声？某个字段在故障发生时是否显著降低了故障原因的不确定性？

看团队沟通，你会问：我发出的信息是否足以让对方减少不确定性？我是不是用了对方没有共享编码表的术语？我有没有加入必要冗余来抵抗噪声？

看学习，你会问：这篇材料带来的新信息在哪里？它是在重复我已知的东西，还是在减少我的关键不确定性？我是否把高熵概念压缩成了可复用的内部编码？

这就是封装的意义。一个公式经过五层学习后，不再停留在纸面上，而变成你的判断工具。

## 硬科学学习的特殊建议

第一，别从数学开始。

这句话不是说数学不重要。信息论当然是数学。没有概率、对数、期望、极限、证明，你无法真正深入。

但数学是压缩后的描述，不一定是最好的学习入口。一个定义可能只占一行，却压缩了几十个例子、多个历史问题和一整套直觉。如果你没有这些直觉，直接读定义就像直接读压缩文件的二进制内容。它确实包含全部信息，但对人类学习者不友好。

入口应该是直觉加代码实验。先看几个具体序列。先算出全 0 的熵是 0，均匀二元的熵是 1，四元均匀的熵是 2，偏斜分布的熵低于均匀分布。先让你的手和眼睛知道这个概念怎么动。然后再回到公式，每个符号都会有位置。

第二，Agent 写代码演示，你做判断。

在 Agent 时代，硬科学学习的效率可以大幅提高。过去你读到一个概念，想做实验，需要自己写样例、查语法、处理边界、画图。现在你可以让 Agent 为每个概念生成最小演示。

比如：

```text
用脚本演示二元分布 p 从 0 到 1 时熵如何变化，输出表格。
写一个脚本比较固定长度编码和按频率设计的变长编码平均长度。
模拟一个有 5% 翻转概率的二进制信道，展示重复 3 次投票如何降低错误率。
```

你的任务不是盲信代码。你的任务是读代码、改参数、预测结果、检查输出是否符合直觉。Agent 负责降低实验成本，你负责保持判断权。

第三，接受“先粗糙再精确”。

硬科学学习中，初始理解经常是错的。你一开始可能以为“熵就是混乱”。后来你会修正：熵是概率分布的不确定性。你一开始可能以为“信息量大的事件更重要”。后来你会修正：小概率事件信息量大，但重要性还取决于目标和损失函数。你一开始可能以为“压缩就是删除重复”。后来你会修正：压缩是利用可预测结构，重复只是其中一种结构。

这些错误不是失败，而是正常路径。

```text
先有一个粗糙直觉 -> 用例子暴露问题 -> 用公式修正 -> 用边界测试 -> 再封装
```

这和软件开发很像。你不会一开始就写出最终架构。你先写最小版本，跑起来，发现边界，重构抽象，最后沉淀接口。硬科学学习也是这样，只不过你重构的是自己的概念系统。

## 总结

硬科学的学习路线和编程不同。

学编程语言或框架，你通常已经有强基线，可以直接做差异映射。学信息论这样的硬科学，你一开始没有足够接近的旧框架，类比也很容易失真。所以第一步不是强行映射，而是先建粗糙基线：读最薄入口材料，建立词汇表，写临时定义，画一张不完美的映射草稿。

基线一旦就绪，后面的管道就熟悉了。

用 HLT 判断它值得深学；用黄金三问找到相似处、差异处和新基元；用精通五层跑通、拆解、实验、探索边界、完成封装。

信息论最后被压缩成一句可以随身携带的话：

```text
熵 = 我对下一个事件有多不确定；
量化后，就是最优编码下平均需要多少 bit。
```

这不是终点，但它是一个真正的入口。你从这里继续走，编码、噪声、信道容量、互信息、压缩和学习，都会开始连成一张网。

学完一个对象还不够。下一章，我们把这些动作收束成个人系统：怎样按季度、月、周、日运行这条管道，让压缩、质检和复盘变成可重复的习惯。
