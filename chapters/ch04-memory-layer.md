# 第四章：记忆层的压缩——Anki、SQ3R 和 Active Recall

## 开篇：你记住了，但你真的理解了吗

有一天，你打开 Anki，突然被一个数字吓到：8000 张卡片。

这不是一个小数字。它代表你过去几个月甚至几年里的认真。你读过的书、看过的论文、刷过的课程、整理过的概念，都变成了一张张正反面卡片。每天早上，你花 45 分钟复习。手机上刷一轮，电脑上刷一轮。绿色按钮、红色按钮、下次复习时间，一切都很有秩序。

你确实记住了很多东西。

看到“TCP 三次握手为什么不是两次”，你能说出答案。看到“梯度下降的学习率过大可能怎样”，你能说出震荡。看到“CAP 定理中的 P 是什么”，你能说出网络分区容忍性。你甚至能在别人刚问出问题时，就条件反射般把卡片背面的句子吐出来。

可是某个瞬间，你会感到不对劲。

你并没有觉得自己更懂 TCP。你还是很难判断一个线上连接异常到底是握手问题、重传问题、拥塞问题，还是应用层超时。你并没有真的更懂梯度下降。别人把优化器从 SGD 换成 Adam，再把 batch size 改大，你仍然只能说一些泛泛的原则。你并没有真的更懂分布式系统。面对一个真实系统的延迟、故障和一致性取舍，你还是不知道该从哪里下手。

你只是更擅长回忆卡片背面的答案。

这就是记忆工具最容易制造的错觉：它让你拥有稳定的提取能力，却不保证你拥有更深的理解能力。你能在正确提示下提取一个答案，不等于你能在变化条件下做判断。你能背出一个定义，不等于你拥有一个可运行的内部模型。

记忆工具解决的是保持率，不是理解深度。

保持率当然重要。遗忘是真实存在的。人脑不是硬盘，读过的东西如果没有提取，就会变模糊、变形、消失。Anki、Active Recall、SQ3R、Cornell 笔记这些方法，都是为了对抗遗忘，或者逼迫你从输入转向输出。

但它们不是同一类问题的终点。

从压缩论的角度看，理解不是把更多内容塞进脑子，而是把复杂信息压缩成更短、更稳、更能迁移的结构。记忆工具可以固定这些结构，让它们长期可用；但如果你放进去的本来就是未经压缩的原文、孤立事实、考试答案，那工具只会帮你更牢地记住这些浅层碎片。

这一章我们不否定记忆工具。相反，我们要把它们放回正确的位置：它们是压缩管道中的“固化层”和“初级输出层”，不是理解本身。你要先知道它们擅长什么，不擅长什么，才不会把“我记得”误认为“我懂了”。

## 间隔重复与 Active Recall：把已压缩的东西长期保存

### 原理

间隔重复的起点，是一个很朴素的事实：你会忘。

你今天读完一段内容，觉得很清楚。明天还能想起大半。三天后只剩一些关键词。一周后，如果没有再碰，脑子里可能只剩“我好像读过”。这就是遗忘曲线描述的现象：新信息进入大脑后，如果不被再次提取，它的可访问性会快速下降。

间隔重复的做法，是在你快要忘掉之前，把它重新提取一次。

太早复习，浪费。你明明还记得，只是在重复看一遍，刺激不够强。太晚复习，也浪费。你已经完全忘了，又接近重新学习。最好的时间点是在“有点吃力但还能想起来”的边缘。这个时候，提取会给记忆留下更强的痕迹。

Active Recall，也就是主动回忆，强调的不是“再看一遍”，而是“合上书说出来”。你不要盯着答案熟悉它，而要在没有答案的情况下生成答案。因为真正增强记忆的，不是信息再次进入眼睛，而是你从脑中把它取出来的动作。

Anki 把这两件事自动化了。它根据你每次回答的难度，安排下一次出现的时间。你答得轻松，间隔拉长；你答错，间隔缩短。于是，你每天面对的不是全部 8000 张卡，而是系统估计“今天最该被提取”的那一小部分。

这是一套非常强的保持率系统。

### 操作

如果你要使用 Anki 或其他间隔重复工具，关键不是“多做卡”，而是“做什么样的卡”。

一张好的卡片应该小而清楚。正面给出一个明确提示，背面给出一个你可以在几秒到几十秒内判断对错的答案。比如：

```text
问：为什么学习率过大可能导致 loss 不下降？
答：因为每次参数更新跨过最优区域，在损失曲面两侧来回震荡，甚至发散。
```

这比“学习率是什么”好一点，因为它要求你说出机制，而不是只说定义。

但还可以更好：

```text
问：在同一模型和数据上，把学习率从 1e-4 改到 1e-2，训练曲线可能出现什么变化？为什么？
答：loss 可能剧烈震荡或发散，因为更新步长过大，参数越过低损失区域；如果优化器有自适应缩放，现象可能被缓和，但仍要看梯度尺度和 batch size。
```

这张卡片里出现了“参数”。你不是只回忆一句定义，而是在一个条件变化下做预测。它已经接近理解质检。

做卡时可以遵守三个规则。

第一，用自己的话。不要复制原文。复制原文是在记字符串，不是在固化理解。你至少要把内容改写成自己能自然说出口的句子。

第二，放入结构，不放入整段。流程、因果链、对比表、最小代码片段，都比长段落更适合进入卡片。

第三，把卡片放在学习之后，而不是学习之前。读到一句话就立刻制卡，常常会制造大量没有经过消化的碎片。更稳的做法是：先跑通、拆解、做一个小实验，再把关键结论做成卡。

### 压缩论分析

Anki 是压缩结果的存储优化器，不是压缩算法本身。

这句话很重要。

你把一个复杂概念学懂之后，可以把它压缩成一句话、一张图、一个例子、一个参数变化预测。Anki 负责让这个压缩结果不要丢。它像一个冷藏柜，可以延长食物保存时间，但它不会把原料自动做成菜。

如果你放进去的是已经压缩好的知识，它就非常有用。

比如你学了数据库索引。你不只是记住“B+ 树适合范围查询”，而是亲手比较过哈希索引和 B+ 树，知道为什么叶子节点链表让范围扫描更顺，知道页分裂会带来写入成本。最后你把结论压缩成卡片：

```text
问：为什么 B+ 树比哈希索引更适合范围查询？
答：因为 B+ 树按 key 有序，叶子节点可顺序遍历；哈希索引只支持按哈希定位单点，破坏了 key 的相邻关系。
```

这时 Anki 固化的是一个结构。

但如果你只是把教材原文贴进去：

```text
B+ 树是一种多路平衡查找树，其非叶子节点只存储索引，叶子节点存储数据，并通过链表连接……
```

然后每天背诵它，你只是训练自己回忆一段长字符串。你可能能在考试中写出定义，却未必能判断一个具体查询为什么没走索引，或者为什么某个联合索引在某个条件下失效。

压缩的质量决定记忆的价值。

未经压缩的内容进入 Anki，只会变成长期负债。卡片越多，复习越重；复习越重，你越没有时间做真正的实验和理解；越没有理解，你越依赖卡片维持“我还在学习”的感觉。

### 与精通五层的关系

在精通五层里，Anki 最好的位置不是 Level 1，也不是 Level 2，而是在 Level 3 之后。

Level 1 是跑通。你知道一个东西能工作。

Level 2 是拆解。你知道输入、处理、输出大概怎么连接。

Level 3 是参数实验。你改一个条件，先预测，再验证。到了这里，你才开始拥有因果理解。

Anki 应该服务于这个因果理解。你做完一个参数实验，发现“batch size 变大后，梯度估计更稳定，但泛化可能变化，学习率也常常要重新调”。这时你把关键结论做成卡片，未来反复提取。这样记住的不是术语，而是经过验证的判断。

不要用 Anki 替代 Level 3。

如果你没有改过参数，只是背“学习率太大震荡，太小收敛慢”，这句话是别人的结论。你当然可以记住，但它在你脑中没有根。等你遇到学习率预热、余弦退火、自适应优化器、梯度裁剪时，这句话就会变得太粗糙，帮不上忙。

一个简单的检查问题是：

```text
这张卡片有没有可以改变的参数？
```

如果有，你能不能预测改变后会发生什么？

比如“HTTP 2 相比 HTTP 1.1 的优势是什么”是一张普通卡。“当一个页面有 100 个小资源，在高延迟网络下，HTTP 2 为什么可能比 HTTP 1.1 更快”就更好，因为它包含资源数量、延迟、多路复用这些参数。

每张 Anki 卡片最好都有自己的"参数"。如果你不能对这张卡的内容做改参数预测，它可能是一张伪理解卡。

这个原则在 Anki 的使用中格外重要，也是它最容易翻车的地方。你放进去的如果不是经过压缩的信息，记忆工具只是在帮你更牢固地记住未理解的东西。

一个常见的国内场景是准备面试时刷"八股文"。很多人会把网上面经里常见的题目做成卡片：

```text
正面：HashMap 和 ConcurrentHashMap 的区别
反面：HashMap 线程不安全，ConcurrentHashMap 线程安全，用分段锁和 CAS 实现
```

这张卡片背熟了，面试时能流畅背出来。但如果面试官换一个角度问：

"如果把 ConcurrentHashMap 的并发级别改成 1，和用 Collections.synchronizedMap 有什么区别？"

很多人会卡住。不是因为不知道分段锁的概念，而是因为卡片里只存了结论，没有存"为什么分段锁比全表锁好"的因果结构。

这个例子不是反对背八股文。事实是，如果能把每个知识点过一遍精通五层——不是只背结论，而是跑通一个 demo、拆解锁的粒度、改参数看看性能变化——那么八股文就是理解的自然产出，而不是替代理解的记忆工作。

但大多数人没有时间做这一步。于是他们直接跳到了 Anki 阶段，把未经压缩的面试结论塞进卡片。结果记住了越来越多的表面答案，却越来越难回答变化型问题。

这不是 Anki 的问题。是用错了时机。

### 典型误用

最常见的误用，是用 Anki 记公式，却不理解公式的推导和适用条件。

你把贝叶斯公式做成卡片：

```text
P(A|B) = P(B|A)P(A) / P(B)
```

你每天复习，永远不会忘。可是当一个真实问题出现时，你仍然不知道哪个是先验，哪个是似然，哪个是证据。你也不知道为什么低基率事件会让直觉判断严重偏差。

这时，Anki 让你考试及格，也让你以为自己懂了。

另一个误用，是把卡片数量当成学习进度。今天新增 80 张，感觉很充实。但如果这 80 张只是原文切片，你实际上给未来的自己增加了复习债务。你会在三个月后每天被这些卡片追着跑，最后只好点“简单”快速逃离。

真正好的 Anki 使用者，不是卡片最多的人，而是敢删卡的人。凡是不能帮助你保持结构、因果、边界和封装的卡片，都应该被合并、重写或删除。记忆工具应该让理解变得可持续，而不是把学习变成一份永远还不完的贷款。

## SQ3R：把阅读从输入改造成第一次压缩输出

### 原理

SQ3R 是一种经典阅读法，五个字母分别代表 Survey、Question、Read、Recite、Review。

Survey 是浏览。你先看标题、目录、小标题、图表、摘要、结论，快速知道这篇文章或这一章大概在讲什么。

Question 是提问。你把标题和小标题改写成问题。比如“小批量梯度下降”变成“小批量为什么比全量或单样本更常用？”

Read 是阅读。带着问题读正文，而不是被动让文字流过眼睛。

Recite 是复述。读完一小段后合上书，用自己的话说出刚才的内容。

Review 是回顾。隔一段时间后重新检查问题、复述和笔记，确认自己是否还能抓住结构。

这个方法的优点，是它把阅读从“看见”改成“生成”。你不是坐在那里等作者把知识倒进来，而是先预判结构、提出问题、再用自己的语言输出。只要你认真执行，它一定比从第一页线性读到最后一页更主动。

### 操作

假设你要读一篇关于向量数据库的技术文章。

第一步 Survey。不要急着读正文。先扫标题、摘要、目录、配图、代码块和结论。你可能会得到一个粗骨架：

```text
向量数据库解决相似性搜索问题
核心是 embedding + ANN 索引
常见索引有 HNSW、IVF、PQ
工程上还要处理过滤、更新、召回率和延迟
```

这一步只需要几分钟。它的目标不是理解细节，而是先知道地形。

第二步 Question。把骨架改成问题：

```text
为什么普通数据库索引不适合向量相似性搜索？
HNSW 为什么能加速近邻搜索？
召回率和延迟之间怎么取舍？
向量检索加 metadata filter 时为什么会变复杂？
```

问题会改变你的注意力。没有问题时，你会平等地读每一句。带着问题时，你会自动寻找答案、证据和例外。

第三步 Read。开始读正文。每读一节，都问：这一节在回答哪个问题？有没有引入新的问题？作者有没有给出条件和边界？

第四步 Recite。合上原文，用自己的话说。不要看着句子改写，那仍然是依赖输入。你可以说：

```text
HNSW 像是在高维空间里建多层导航图。上层边少，负责快速跳到大致区域；下层边多，负责局部精细搜索。所以它不是保证找到精确最近邻，而是在速度和召回之间做近似。
```

第五步 Review。隔几个小时或第二天，再看你提出的问题，尝试不看原文回答。如果答不上来，回到原文定位，不要从头重读。

### 压缩论分析

SQ3R 里最值得关注的是两个动作：Survey 和 Recite。

Survey 是通读全文前的骨架抽取。本质上，它是一种采样压缩。

你还没有能力压缩全文，因为你还没读。但你可以先采样：标题通常暴露主题，目录暴露结构，图表暴露关键关系，结论暴露作者想让你带走的东西。你用这些高信号位置，先构造一个低分辨率地图。

这一步非常节省认知资源。

很多人读技术文档会犯一个错误：从第一行开始平均用力。结果在背景介绍上消耗太多精力，到真正关键的设计取舍时已经疲劳。Survey 的价值就是告诉你：哪里可能重要，哪里可以快读，哪里需要慢下来。

Recite 则是第一次压缩输出。

你刚读完一段内容，脑中还有很多细节。Recite 要求你删掉一部分细节，保留结构，用自己的话重新生成。这就是压缩操作：删细节，保关系；删例子，保机制；删修辞，保判断。

但 Recite 有一个严重问题：它缺乏压缩质量检验。

你能复述，不代表你复述对了。你可能漏掉了关键条件，也可能把作者的谨慎表达改成了绝对结论。比如作者说“在某些高延迟、多资源请求场景下，HTTP 2 的多路复用可能减少队头阻塞影响”，你复述成“HTTP 2 一定比 HTTP 1 快”。这就是压缩失真。

所以 SQ3R 的 Recite 只是压缩，不是验证。它让你第一次把输入变成自己的输出，但还没有证明这个输出能预测、能迁移、能经受反例。

### 与精通五层的关系

SQ3R 主要是 Level 1 到 Level 2 的工具。

Survey 接近 Level 1 前的结构预扫。你还没有跑通，但你先知道这片知识大概有哪些模块、顺序和出口。它让你更快建立“这个东西在做什么”的初步感受。

Question 和 Read 帮你进入 Level 2。你开始拆解作者的结构：问题是什么，方案是什么，证据是什么，结论是什么。你不再只记内容，而是在追踪输入、处理和输出。

Recite 则是 Level 2 后的自我检验。你能不能不看原文说出每一部分在做什么？能不能把一节内容压缩成一个因果链或一个对比？

但 SQ3R 本身很少进入 Level 3、Level 4 和 Level 5。

它不会自动要求你改参数。读完一篇缓存淘汰策略文章，SQ3R 可以帮你说清 LRU、LFU、FIFO 的区别，但不会逼你回答：“如果访问模式从均匀随机变成热点突增，哪个策略会失效？”这需要 Level 3 的参数实验。

它也不会自动要求你制造失败。读完一篇微服务文章，SQ3R 可以帮你复述服务拆分原则，但不会逼你设计一个拆分过度导致调用链膨胀的反例。这是 Level 4。

它更不会自动把知识挂到你的旧框架里。那是 Level 5 的封装任务。

所以，SQ3R 是很好的阅读启动器，不是理解终点。

### 典型误用

SQ3R 的第一个误用，是只做 Survey 和 Read，不做 Recite。

很多人喜欢“先浏览再阅读”，但读完之后不合上书复述。这样 SQ3R 就退化成带预览的普通阅读。真正改变学习性质的是 Recite，因为它强迫你输出。没有输出，压缩就没有发生。

第二个误用，是把 Question 做成形式主义。

你把小标题“自注意力机制”改成“什么是自注意力机制？”这当然是一个问题，但太弱。更好的问题应该逼近机制和边界：

```text
为什么自注意力能在不使用循环结构的情况下建模序列关系？
它的计算量为什么会随序列长度平方增长？
当序列特别长时，这个机制哪里会先出问题？
```

问题质量决定阅读质量。

第三个误用，是把 SQ3R 用在不适合的输入上。

SQ3R 假设你面对的是结构良好的文本：书、文章、课程讲义。它们通常有标题、段落、层次和作者意图。可如果你面对的是一坨遗留代码、一次线上事故、一份散乱的系统设计讨论，SQ3R 的适用性就会下降。

代码不一定按解释顺序组织。系统设计文档可能省略上下文。线上事故里最关键的信息常常藏在日志、指标和时间线里，而不是标题下面。面对这些非结构化输入，你仍然可以借用 Survey 的思想先扫结构，但不能指望五步法完整接管。你需要更强的拆解、实验和边界验证。

SQ3R 能帮你读得更主动，但不能替你完成理解质检。

## Cornell Note-Taking：把笔记压缩成一句话

### 原理

Cornell 笔记法的页面通常分成三块。

右栏是主笔记区。你在这里记录课堂、书籍或会议中的主要内容。它可以比较详细，包括定义、例子、图示、推导、代码片段。

左栏是关键词或问题区。右栏写完后，你回头在左栏提炼关键词，或者写出能覆盖右栏内容的问题。它像一组索引，帮助你以后遮住右栏进行回忆。

底部是总结区。你用很短的一段话，最好是一两句话，总结这一页最重要的内容。

这套格式看起来简单，但它隐含了三次处理。

第一次是记录。你把输入留下来。

第二次是索引。你从输入中抽出关键词和问题。

第三次是总结。你把整页内容压缩成一个最小输出。

很多人以为 Cornell 的价值在页面整洁。其实不是。它真正的价值在于：它不允许你只做线性记录。它逼你在笔记旁边建立提取入口，并在底部做一次高强度压缩。

### 操作

假设你在听一节关于缓存一致性的课。

右栏可以记录主要内容：

```text
缓存提升读取速度，但会引入数据不一致。
常见策略：Cache Aside、Read Through、Write Through、Write Back。
Cache Aside 中应用先读缓存，未命中再读数据库，并回填缓存。
写入时通常先更新数据库，再删除缓存。
问题：删除缓存失败会导致脏数据；并发读写也可能把旧值回填。
```

左栏不要只写“缓存一致性”。那太粗。你可以写成问题：

```text
为什么缓存会破坏一致性？
Cache Aside 的读写路径是什么？
为什么常见做法是更新数据库后删除缓存，而不是更新缓存？
并发读写时旧值如何回填？
```

底部总结必须更短：

```text
缓存一致性的核心不是“让缓存永远正确”，而是在读写路径、失败重试和过期策略之间控制不一致窗口。
```

这一句话就是压缩输出。

它不包含所有策略名，不包含每个异常路径，但它抓住了主结构：缓存一致性是控制不一致窗口，而不是追求绝对同步。未来你读到延迟双删、binlog 订阅、版本号校验，都可以挂到这句话下面。

Cornell 的使用关键，是顺序不要偷懒。

课堂或阅读中可以先写右栏。结束后，不要立刻合上本子。花三分钟补左栏问题，再花一分钟写底部总结。最后遮住右栏，只看左栏问题，尝试回答。答不出来的地方，说明右栏记录还没有进入可提取状态。

### 压缩论分析

Cornell 的底部总结，是非常小但非常重要的压缩练习。

一整页笔记可能有几百字。底部只给你几行。空间限制会逼你选择：什么是中心？什么只是例子？什么是条件？什么是结论？什么可以删掉？

这和真正的理解很接近。理解不是保留全部，而是知道哪些不能删。

但 Cornell 仍然只是压缩练习，不是压缩验证。

你写出一句漂亮总结，并不代表它正确。你可能总结得过宽，也可能总结得过窄。比如你把缓存一致性总结成：

```text
缓存一致性的关键是写数据库后删除缓存。
```

这句话并非完全错，但它太像操作步骤，缺少更上层的结构。它没有说明为什么要这样做，也没有覆盖失败、并发和不一致窗口。它看起来简洁，实际压缩掉了关键机制。

Cornell 给了你输出格式，但没有给你质量检验。

格式只能减少混乱，不能保证洞察。一个人可以用 Cornell 写出非常清楚的浅笔记：右栏复制课件，左栏写几个名词，底部写“本页介绍了缓存一致性的基本概念”。这当然整齐，但没有压缩价值。

真正有价值的底部总结，应该能承受追问。

比如：

```text
如果缓存过期时间从 5 分钟改成 5 秒，这个总结还成立吗？
如果写数据库成功但删缓存失败，这个总结是否能解释风险？
如果系统从单机缓存变成多级缓存，这个总结要不要改？
```

当你开始这样追问，Cornell 就从笔记格式进入了理解质检。

### 与精通五层的关系

Cornell 的右栏主要服务 Level 1 和 Level 2。它帮你记录现象、概念、步骤和结构。右栏写得好，你以后能快速恢复当时的输入。

左栏问题也服务 Level 2。它把线性内容改造成可提取的结构。你不再按原文顺序找信息，而是通过问题进入知识。

底部总结最接近 Level 5。

Level 5 的目标，是把一个知识压缩成一句话，并挂到已有框架上。Cornell 每一页都要求你写总结，所以它天然是 Level 5 的训练场。你可以把每页底部当成一个小型封装：

```text
这一页到底想让我以后记住什么？
它和我已经知道的哪个模型相连？
如果只能留一句话，我留哪一句？
```

但如果你只写总结，仍然可能停留在漂亮话。要把它拉回 Level 3，你需要加一个问题：

```text
如果改一个参数，这个总结还成立吗？
```

比如你的底部总结是：

```text
批处理通过摊薄固定开销提高吞吐，但会增加单个请求等待时间。
```

现在改参数：batch size 从 32 变成 1024，会怎样？请求到达率降低会怎样？单次处理时间不是线性增长会怎样？一旦你能回答这些问题，这句总结就不只是封装，而是连接到了因果模型。

Cornell 可以训练 Level 5，也可以通过参数追问回到 Level 3。它最好的用法不是“记得更整齐”，而是“每页都逼自己做一次最小封装，再检查这个封装是否能预测变化”。

### 典型误用

Cornell 最常见的误用，是只写右栏和左栏，从来不写底部总结。

这样它就退化成线性笔记加关键词。页面看起来仍然像 Cornell，但最关键的压缩动作消失了。你只是把内容搬到了纸上，再在旁边贴了几个标签。

第二个误用，是底部总结写得太空。

比如：

```text
本页主要讲了微服务的优缺点。
```

这不是总结，只是标题复述。它没有告诉未来的你任何判断。更好的写法是：

```text
微服务用服务边界换取独立部署和团队自治，但会把复杂度从进程内部转移到网络、数据一致性和运维系统。
```

这句话有取舍、有机制、有边界，未来还能被参数追问。

第三个误用，是把 Cornell 当成课堂速记法，而不是学习后的加工法。

上课时你当然要先记下足够信息。但 Cornell 真正发生作用，往往是在课后五分钟。那时你不再忙着跟上老师，而是可以回头问：这一页的主线是什么？左栏问题能不能覆盖右栏？底部总结是不是一句可迁移的话？

如果没有这一步，笔记只是存档。做完这一步，笔记才开始变成压缩。

## 总结：记忆工具在压缩管道中的位置

现在我们可以把这几种方法放回同一条管道里看。

- Anki / 间隔重复：压缩操作是把已经形成的结构、因果链、判断和例子做成可提取卡片；质检机制主要是能否主动回忆，但它不验证理解是否正确；适用层级最好在 Level 3 之后，用来固化经过参数实验验证的结论；与精通五层的关系是保存理解产物，不替代理解过程。

- Active Recall：压缩操作是合上资料后主动生成答案；质检机制是和原答案对照，但容易只检查表面一致；适用层级覆盖 Level 1 到 Level 2，也可以辅助 Level 3 的预测；与精通五层的关系是把“看懂”变成“说出”，但还要继续接受参数和边界检验。

- SQ3R：压缩操作是 Survey 的采样压缩和 Recite 的第一次复述压缩；质检机制较弱，主要靠 Review 发现遗忘，不能充分发现歪曲；适用层级是 Level 1 到 Level 2；与精通五层的关系是帮助你建立阅读骨架和初步拆解，不负责深层因果。

- Cornell Note-Taking：压缩操作是右栏记录、左栏索引、底部一句话总结；质检机制取决于你是否用问题和参数变化追问总结；适用层级从 Level 1 到 Level 2 起步，底部总结可训练 Level 5；与精通五层的关系是提供封装练习，但需要 Level 3 的参数问题来检验质量。

这张表背后的结论很简单：

记忆工具只能固定已压缩的内容，不能创造理解。

你可以用 Anki 保持一个因果模型，但 Anki 不会自动帮你建立因果模型。你可以用 SQ3R 让阅读更主动，但 SQ3R 不会自动帮你做参数实验。你可以用 Cornell 写出一句漂亮总结，但漂亮总结不等于真实理解。

顺序很关键。

先用精通五层学透：跑通现象，拆解结构，改变参数，制造失败，最后一句话封装。然后再用记忆工具把关键产物固化下来。这样，你记住的是经过压缩和质检的知识。

如果顺序反了，你会变成一个知道很多但理解很浅的人。

你会有 8000 张卡片，每天复习 45 分钟，能答出很多标准答案，却在真实问题前缺少判断力。那不是因为你不努力，而是因为你把保持率工具放在了理解工具的位置上。

记忆是必要的。没有记忆，理解无法长期存在。

但记忆不是终点。它只是让真正理解过的东西，不那么快从你手里滑走。

下一章，我们继续看另一类经典框架：它们不直接解决记忆保持，而是试图描述理解的层级和能力成长。但描述层级，仍然不等于完成压缩质检。
