## 写作风格

- 给出可执行的周期设计、日志模板、节点判断
- 目标字数：6,000字

## 章节大纲

### 开篇（~500字）

你学完了前三卷的所有工具，知道了HLT、差异驱动、精通五层、Agent辅助、以及各种经典方法在压缩管道中的位置。

现在的问题不是"还有什么方法"——而是"我怎么让它长期运行？"

### 学习周期设计（~2,000字）

**季度级 → 月级 → 周级 → 日级：**

**季度扫描（每3个月一次，2小时）：**
- 做HLT评分：列出10个当前考虑的学习方向，评分排序
- 淘汰低于75的，锁定1-2个深度目标
- 复盘过去3个月的学习：哪些HLT评分变了？为什么？
- 输出：季度学习计划（1-2个深度目标 + 2-3个轻量了解）

**每月规划（30分钟）：**
- 对季度目标做差异映射：和已知框架的挂钩点
- 拆解为"本周完成Level 1-2，下周完成Level 3..."
- 输出：月度学习里程碑

**每周执行（每天30-60分钟，看精力）：**
- 按精通五层的顺序推进
- 至少一次Level 3参数实验（这是硬性要求，不可跳过）
- 输出：每周一篇学习日志（预测+验证+反思）

**每日最小行动：**
- 最小动作：5分钟做一个预测（"今天读的这段，如果条件X变成Y，结果会怎样？"）
- 不追求大量，追求连续——每天一个预测，一个月30个，数据量足够分析

### 学习日志模板（~1,500字）

**精通五层日志——每层有专门的记录格式：**

Level 1-2 日志（一次性，记录结果即可）：
```
主题：[概念名]
跑通结果：[成功/失败，关键输出]
拆解完成：[数据流图已画/未完成]
卡点：[如果有，记下]
```

Level 3 日志（这是最关键的日志——每次实验一条）：
```
参数：[改了什么]
预测：[改之前写下]
实际结果：[改之后看到]
判定：[预测正确/错误]
如果错误➡原因：[缺了什么因果链]
```

Level 4 日志：
```
失败场景：[构造了什么]
实际失效模式：[发生了什么]
根本原因：[哪一层出问题]
```

Level 5 日志：
```
一句话封装：[]
挂载目标：[和什么框架相同/不同]
```

**月度回顾模板：**
```
Level 3 实验次数：[]
预测正确率：[]
发现的因果缺口：[列出]
下月调整：[学习方向修正/方法调整]
```

### 什么时候"可以了"？——每个阶段的退出信号（~1,000字）

**Level 1 退出：** 能回答"它做了什么"。不用讲原理，看到结果说清楚。
**Level 2 退出：** 数据流图画完了，每个节点三问（输入/处理/输出）有答案。
**Level 3 退出：** 连续三个参数变化你都预测正确。任何一个预测错误 → 回到Level 2修正。
**Level 4 退出：** 至少制造了一个失败场景，理解了根本原因。
**Level 5 退出：** 一句话写出来了自己觉得"对"。

**系统的停止条件：** 当你不再需要这个系统来判断"我学会了没有"的时候——你的直觉已经能告诉你"我懂了"还是"我只是觉得我懂了"。系统的作用是训练这个直觉，不是永远绑住你。

### 整合其他工具（~800字）

**Anki：** 在Level 3通过后，把因果模型的关键输出做成卡片。不在Level 2之前做卡片。
**Zettelkasten：** 在Level 5之后建卡片，标注理解层级。
**思维导图：** 在Level 5之后画，作为压缩输出的可视化。
**SQ3R/Adler：** 在进入新领域的第一步用（采样压缩），快速确定重点区域。

**关键原则：** 先压缩（五层），再组织（ZK），再固化（Anki），再展示（导图）。顺序颠倒，效果打折。

### 总结（~300字）

学习系统不是一个App、不是一个课程、不是一个书单。它是一个可重复运行的流程——HLT选方向→差异驱动挂框架→精通五层质检→Agent辅助加速。把这个流程跑三遍，它就变成了你的习惯。
