# Codex: 生成第四章「记忆层的压缩——Anki、SQ3R和Active Recall」

## 指令

你是一个中文科技畅销书作家。请根据以下大纲撰写第四章。

## 输出

写入文件：`/root/Projects/compress-to-understand/chapters/ch04-memory-layer.md`

## 写作风格

- 中文流畅，非翻译腔，用"你"建立对话感
- 每个方法：原理 → 操作 → 压缩论分析 → 与精通五层的关系 → 典型误用
- 目标字数：6,000字

## 章节大纲

### 开篇（~600字）

场景：你发现自己的Anki卡片积累了8000张，每天复习花45分钟，但你并没有觉得自己更"理解"那些知识。你只是更擅长回忆卡片背面的答案。引出核心论点：记忆工具解决的是保持率，不是理解深度。

### 间隔重复 + Active Recall（~1,500字）

**原理：** Ebbinghaus遗忘曲线 → 最优间隔提取 → 长期保持。
**压缩论分析：** Anki是压缩结果的存储优化器，不是压缩算法本身。你把已经压缩好的知识（自己的话/流程图/代码）放进Anki固化。如果把未经压缩的原文放进去，你只是在记住一个长字符串。
**常见误用：** 用Anki记公式但不理解公式的推导和适用条件。Anki可以让你考试及格，但让你以为自己懂了。
**与精通五层的关系：** Anki的最佳使用位置是在Level 3（参数实验）之后——你验证了因果理解，把因果模型的关键输出卡片化。不要用Anki替代Level 3。
**建议：** 每张Anki卡片应该有自己的"参数"——如果你不能对这张卡片的内容做改参数预测，它可能是一张伪理解卡片。

### SQ3R（Survey→Question→Read→Recite→Review）（~1,500字）

**原理：** 五步阅读法。
**Recite的压缩论分析：** Recite是第一次压缩输出——你要用自己的话复述刚刚读到的内容。这是压缩操作（删细节保结构）。但缺乏压缩质量检验——复述可能是歪曲的、遗漏关键的。
**Survey的压缩价值：** 通读全文前的骨架抽取——本质上是一种"采样压缩"。先快速获取文档结构，再决定在哪里投入细粒度压缩资源。
**与精通五层的关系：** SQ3R是Level 1-2的工具。Survey≈跑通前的结构预扫，Recite≈拆解后的自我检验。它不涉及Level 3-5的深度。
**局限：** 假设你在读一本书/文章这种结构良好的输入。对于非结构化输入（代码、系统设计文档），SQ3R的适用性下降。

### Cornell Note-Taking（~1,500字）

**原理：** 右栏（主笔记）→ 左栏（关键词/问题）→ 底部（总结）。
**压缩论分析：** 底部总结是全书最小的单次压缩输出——你被迫在一句话中提取要点。这是压缩练习，不是压缩验证。Cornell给了你"输出格式"但没有给"质量检验"。
**与精通五层的关系：** Cornell的底部总结可以作为Level 5（一句话封装）的训练场。每次记完笔记，写出底部总结——然后问自己"如果改一个参数，这个总结的内容还成立吗？"就进入Level 3了。
**常见误用：** 只写了右栏和左栏，从来没写过底部总结。那Cornell就退化为线性笔记。
